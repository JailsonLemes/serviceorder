Sistema de Pagamento

Modelo escrow com integração Stripe (cartão) e Gerencianet/Efí (PIX). Comissão da plataforma capturada na autorização. Liberação automática após aprovação da entrega.

1. Camada de Abstração de Gateways

O sistema nunca chama Stripe ou Gerencianet diretamente no código de negócio. Existe uma interface PaymentGateway com implementações específicas por provedor. Isso permite trocar gateways sem tocar na lógica de escrow, comissões ou disputas.

src/payment/gateways/payment-gateway.interface.ts
export interface PaymentGateway {
  // Pré-autoriza: reserva o valor sem debitar (cartão) ou gera cobrança (PIX)
  authorize(input: AuthorizeInput): Promise<AuthorizeResult>;

  // Captura: efetiva a cobrança (muda de reserva para débito)
  capture(providerPaymentId: string): Promise<CaptureResult>;

  // Transfere valor ao freelancer (via conta bancária ou chave PIX)
  transfer(input: TransferInput): Promise<TransferResult>;

  // Estorna total ou parcial
  refund(input: RefundInput): Promise<RefundResult>;

  // Valida assinatura do webhook recebido
  validateWebhookSignature(payload: string, signature: string): boolean;

  // Normaliza evento do provedor para formato interno
  parseWebhookEvent(payload: unknown): WebhookEvent;
}

// Tipos internos normalizados (independente do provedor)
export type WebhookEvent = {
  type:            'payment.authorized' | 'payment.failed' | 'payment.refunded';
  providerPaymentId: string;
  amount:           string;  // string decimal, ex: "3500.00"
  currency:         string;
  metadata:         Record<string, string>;
};

2. Integração Stripe — Cartão de Crédito/Débito

Stripe usa o modelo PaymentIntent com capture_method: 'manual'. Isso separa a autorização (reserva de fundos) da captura (débito efetivo), que é exatamente o que o escrow exige. O cartão nunca é armazenado na plataforma — apenas o token gerado pelo Stripe.js no frontend.

src/payment/gateways/stripe.gateway.ts
import Stripe from 'stripe';
import { PaymentGateway } from './payment-gateway.interface';

export class StripeGateway implements PaymentGateway {
  private stripe = new Stripe(process.env.STRIPE_SECRET_KEY, {
    apiVersion: '2024-06-20',
    timeout:    10000,
  });

  async authorize({ amount, currency, providerToken, orderId, paymentId }) {
    // PaymentIntent com capture_method manual = escrow nativo do Stripe
    const pi = await this.stripe.paymentIntents.create({
      amount:         Math.round(Number(amount) * 100),  // Stripe usa centavos
      currency:       currency.toLowerCase(),          // 'brl'
      payment_method: providerToken,
      capture_method: 'manual',                        // NÃO debita imediatamente
      confirm:        true,
      metadata: {
        order_id:   orderId,
        payment_id: paymentId,
      },
      // Autorização expira em 7 dias se não capturada — Stripe cancela automaticamente
    });

    return {
      providerPaymentId:  pi.id,
      providerChargeId:   pi.latest_charge as string,
      status:             'AUTHORIZED',
      providerMetadata:   pi,
    };
  }

  async capture(providerPaymentId: string) {
    // Efetua o débito real na captura do escrow (freelancer aceitou)
    const pi = await this.stripe.paymentIntents.capture(providerPaymentId);
    return { status: 'CAPTURED', capturedAt: new Date().toISOString() };
  }

  async transfer({ amount, currency, stripeAccountId, orderId }) {
    // Stripe Connect: transfere para a conta conectada do freelancer
    const transfer = await this.stripe.transfers.create({
      amount:      Math.round(Number(amount) * 100),
      currency:    currency.toLowerCase(),
      destination: stripeAccountId,         // conta Stripe do freelancer
      metadata: { order_id: orderId },
    });
    return { providerTransferId: transfer.id, releasedAt: new Date().toISOString() };
  }

  async refund({ providerPaymentId, amount }) {
    const refund = await this.stripe.refunds.create({
      payment_intent: providerPaymentId,
      amount:         amount ? Math.round(Number(amount) * 100) : undefined, // undefined = total
    });
    return { providerRefundId: refund.id, refundedAt: new Date().toISOString() };
  }

  validateWebhookSignature(payload: string, signature: string) {
    try {
      this.stripe.webhooks.constructEvent(
        payload, signature, process.env.STRIPE_WEBHOOK_SECRET
      );
      return true;
    } catch {
      return false;
    }
  }
}

3. Integração PIX — Gerencianet / Efí

O PIX tem fluxo assíncrono: a plataforma gera a cobrança e retorna o QR code. O pagamento chega via webhook quando o usuário paga. A pré-autorização só é confirmada quando o webhook pix.recebido chega com o status correto.

⚠️ Expiração PIX: cobranças PIX expiram em 30 minutos por padrão. Se o freelancer aceitar antes do pagamento PIX ser confirmado, o aceite deve ser bloqueado até a confirmação chegar via webhook.

src/payment/gateways/gerencianet.gateway.ts
import { EfiPay } from 'sdk-node-apis-efi';

export class GerencianetGateway implements PaymentGateway {
  private efi = new EfiPay({
    client_id:     process.env.EFI_CLIENT_ID,
    client_secret: process.env.EFI_CLIENT_SECRET,
    sandbox:       process.env.NODE_ENV !== 'production',
    certificate:   './certs/efi-prod.p12',         // mTLS obrigatório pelo BACEN
  });

  async authorize({ amount, orderId, paymentId, payerName }) {
    // Gera cobrança PIX imediata (cob) com expiração de 30 min
    const cob = await this.efi.pixCreateImmediateCharge({}, {
      calendario: { expiracao: 1800 },              // 30 min em segundos
      valor: { original: amount },                  // string com 2 casas decimais
      chave:     process.env.EFI_PIX_KEY,          // chave PIX da plataforma
      infoAdicionais: [
        { nome: 'Ordem', valor: orderId   },
        { nome: 'Ref',   valor: paymentId },
      ],
    });

    // Gera QR Code para exibir ao usuário
    const qr = await this.efi.pixGenerateQRCode({ id: cob.loc.id });

    return {
      providerPaymentId: cob.txid,
      status:            'PENDING',              // PIX fica PENDING até webhook
      pixQrCode:         qr.qrcode,             // string EMV para copiar e colar
      pixQrCodeUrl:      qr.imagemQrcode,       // URL da imagem do QR Code
      pixExpirationAt:   new Date(Date.now() + 30 * 60000).toISOString(),
    };
  }

  validateWebhookSignature(payload: string, signature: string) {
    // Gerencianet envia mTLS — a validação é feita pelo certificado do servidor
    // O BACEN exige que o endpoint PIX use mTLS com certificado da EFí
    const hmac = createHmac('sha256', process.env.EFI_WEBHOOK_SECRET)
      .update(payload).digest('hex');
    return timingSafeEqual(Buffer.from(hmac), Buffer.from(signature));
  }
}

4. Pagamento Externo (Manual)

Empresas enterprise pagam por boleto, transferência ou invoice. Não há gateway. Um admin registra o recebimento manualmente após confirmação bancária. O fluxo exige aprovação de dois admins para valores acima de R$ 50.000 (four-eyes principle).

src/payment/gateways/manual.gateway.ts
export class ManualGateway implements PaymentGateway {
  async authorize({ amount, orderId, paymentId, referenceCode, approvedByAdminId }) {
    // Four-eyes: valores > 50k exigem segundo admin
    if (Number(amount) > 50000 && !approvedByAdminId)
      throw new BusinessError('SECOND_APPROVAL_REQUIRED');

    // Gera ID interno — não existe provider externo
    const internalId = `MANUAL-${orderId}-${Date.now()}`;

    // Registra no audit_log com referência bancária e adminId
    await auditLog({
      entityType: 'payment', entityId: paymentId,
      action:    'manual.payment_registered',
      actorId:  approvedByAdminId,
      afterData: { amount, referenceCode, internalId },
    });

    return {
      providerPaymentId: internalId,
      status:            'AUTHORIZED',   // imediatamente autorizado (admin confirmou)
    };
  }

  async transfer({ amount, pixKey, orderId, approvedByAdminId }) {
    // Para pagamentos manuais, a transferência é também manual
    // O sistema registra a intenção; o admin processa no banco e confirma
    await queue.publish('manual-transfers', {
      orderId, amount, pixKey,
      requestedBy: approvedByAdminId,
    });
    return { providerTransferId: `PENDING-MANUAL-${orderId}` };
  }
}

5. PaymentService — Orquestração Central

O PaymentService é chamado pelo OrderService nos momentos corretos da FSM. Toda operação financeira acontece dentro de uma transação de banco para garantir consistência entre as tabelas payments, payment_transactions e platform_ledger.

src/payment/payment.service.ts
export class PaymentService {

  // ── 1. AUTORIZAR (empresa paga, freelancer ainda não aceitou) ──────────
  static async authorize(input: AuthorizeInput) {
    const order   = await OrderRepo.findById(input.orderId);
    const feeRate = await PlatformSettings.get('platform_fee_rate'); // snapshot
    const feeAmt  = Decimal.mul(input.amount, feeRate).toFixed(2);
    const freeLrAmt = Decimal.sub(input.amount, feeAmt).toFixed(2);

    return db.transaction(async (trx) => {
      // 1. Cria o registro de pagamento (PENDING) com taxa fixada agora
      const payment = await trx.query(`
        INSERT INTO payments (order_id, company_id, freelancer_id,
          gross_amount, platform_fee_rate, platform_fee_amount,
          freelancer_amount, currency, status, idempotency_key)
        VALUES ($1,$2,$3,$4,$5,$6,$7,$8,'PENDING',$9)
        ON CONFLICT (idempotency_key) DO NOTHING
        RETURNING *
      `, [
        input.orderId,    order.company_id,  order.selected_freelancer_id,
        input.amount,     feeRate,          feeAmt,
        freeLrAmt,        input.currency,   input.idempotencyKey,
      ]);

      if (!payment.rows.length)
        throw new ConflictError('PAYMENT_ALREADY_EXISTS');

      // 2. Chama o gateway correto
      const gateway = resolveGateway(input.method);
      const result  = await gateway.authorize({
        amount:        input.amount,
        currency:      input.currency,
        providerToken: input.providerToken,
        orderId:       input.orderId,
        paymentId:     payment.rows[0].id,
      });

      // 3. Atualiza payment com dados do provedor
      await trx.query(`
        UPDATE payments SET
          status = $1, method = $2, provider = $3,
          provider_payment_id = $4, pix_qr_code = $5,
          pix_expiration_at = $6, authorized_at = NOW()
        WHERE id = $7
      `, [
        result.status,           input.method,      resolveProviderName(input.method),
        result.providerPaymentId, result.pixQrCode, result.pixExpirationAt,
        payment.rows[0].id,
      ]);

      // 4. Registra transação no log imutável
      await TxLog.insert(trx, {
        paymentId: payment.rows[0].id,
        type:      'AUTHORIZATION',
        status:    'SUCCESS',
        amount:    input.amount,
      });

      return { payment: payment.rows[0], ...result };
    });
  }

  // ── 2. CAPTURAR (freelancer aceitou → escrow efetivado) ────────────────
  static async capture(paymentId: string) {
    const payment = await PaymentRepo.findById(paymentId);

    if (payment.status !== 'AUTHORIZED')
      throw new InvalidStateError('PAYMENT_NOT_AUTHORIZED');

    // PIX já é capturado automaticamente quando o usuário paga
    if (payment.method === 'PIX') {
      await db.query(`
        UPDATE payments SET status='CAPTURED', captured_at=NOW() WHERE id=$1
      `, [paymentId]);
      return;
    }

    // Cartão: captura via Stripe (débito efetivado)
    const gateway = resolveGateway(payment.method);
    await gateway.capture(payment.provider_payment_id);

    await db.transaction(async (trx) => {
      await trx.query(`
        UPDATE payments SET status='CAPTURED', captured_at=NOW() WHERE id=$1
      `, [paymentId]);
      await TxLog.insert(trx, {
        paymentId, type: 'CAPTURE', status: 'SUCCESS',
        amount: payment.gross_amount,
      });
    });
  }

  // ── 3. LIBERAR (job dispara após timer de escrow) ──────────────────────
  static async release(paymentId: string) {
    const payment    = await PaymentRepo.findById(paymentId);
    const freelancer = await FreelancerRepo.findById(payment.freelancer_id);

    if (payment.status !== 'CAPTURED')
      throw new InvalidStateError('CANNOT_RELEASE');

    const gateway = resolveGateway(payment.method);
    const result  = await gateway.transfer({
      amount:          payment.freelancer_amount,
      currency:        payment.currency,
      pixKey:          freelancer.pix_key,        // descriptografada na app
      stripeAccountId: freelancer.stripe_account_id,
      orderId:         payment.order_id,
    });

    await db.transaction(async (trx) => {
      await trx.query(`
        UPDATE payments SET status='RELEASED', released_at=NOW() WHERE id=$1
      `, [paymentId]);

      await trx.query(`
        UPDATE orders SET status='COMPLETED', completed_at=NOW() WHERE id=$1
      `, [payment.order_id]);

      // Registra receita da plataforma no ledger
      await trx.query(`
        INSERT INTO platform_ledger (payment_id, entry_type, amount, currency, balance_after)
        VALUES ($1, 'FEE_EARNED', $2, $3,
          (SELECT COALESCE(SUM(amount),0) FROM platform_ledger WHERE entry_type='FEE_EARNED') + $2)
      `, [paymentId, payment.platform_fee_amount, payment.currency]);

      await TxLog.insert(trx, {
        paymentId, type: 'RELEASE', status: 'SUCCESS',
        amount: payment.freelancer_amount,
        providerTransactionId: result.providerTransferId,
      });

      await eventBus.emit('order.completed', { orderId: payment.order_id });
    });
  }
}

6. Tratamento de Webhooks

O webhook é a peça mais crítica de confiabilidade. Deve ser idempotente (mesmo evento processado duas vezes não causa efeito duplo), processado de forma assíncrona (responde 200 imediatamente, processa na fila), e cada entrega tem retry automático com exponential backoff.

src/payment/webhook.controller.ts
@Post('/webhooks/payment/:provider')
async handlePaymentWebhook(req: Request, res: Response) {
  const rawBody  = req.rawBody;  // IMPORTANTE: body raw, não parseado (assinatura HMAC)
  const provider = req.params.provider;
  const signature= req.headers['stripe-signature']
                  ?? req.headers['x-efi-signature'];

  // 1. Valida assinatura HMAC — rejeita silenciosamente se inválido
  const gateway = resolveGateway(provider);
  if (!gateway.validateWebhookSignature(rawBody, signature as string))
    return res.status(401).end();

  // 2. Persiste o webhook raw antes de qualquer processamento
  const webhookId = await db.query(`
    INSERT INTO payment_webhooks (provider, event_type, payload, is_verified)
    VALUES ($1, $2, $3, true)
    RETURNING id
  `, [provider, req.body.type, req.body]);

  // 3. Responde 200 imediatamente (Stripe retenta se não responder em 30s)
  res.json({ received: true });

  // 4. Processa de forma assíncrona na fila com retry automático
  await queue.publish('payment-webhooks', {
    webhookId: webhookId.rows[0].id,
    provider,
    eventType: req.body.type,
    payload:   req.body,
  });
}

// Worker que processa a fila de webhooks
queue.subscribe('payment-webhooks', {
  attempts: 5,
  backoff:   { type: 'exponential', delay: 2000 },  // 2s, 4s, 8s, 16s, 32s
}, async ({ webhookId, provider, payload }) => {
  // Idempotência: verifica se já foi processado
  const wh = await db.query(
    'SELECT processing_status FROM payment_webhooks WHERE id=$1', [webhookId]
  );
  if (wh.rows[0].processing_status === 'PROCESSED') return;

  const gateway = resolveGateway(provider);
  const event   = gateway.parseWebhookEvent(payload);

  switch (event.type) {
    case 'payment.authorized':   // PIX confirmado pelo banco
      await PaymentService.markAuthorized(event.providerPaymentId);
      break;
    case 'payment.failed':       // PIX expirou, cartão recusado
      await PaymentService.markFailed(event.providerPaymentId, event.reason);
      break;
    case 'payment.refunded':     // Estorno confirmado pelo provedor
      await PaymentService.markRefunded(event.providerPaymentId);
      break;
  }

  await db.query(`
    UPDATE payment_webhooks
    SET processing_status='PROCESSED', processed_at=NOW(), payment_id=$1
    WHERE id=$2
  `, [event.paymentId, webhookId]);
});

7. Job de Liberação Automática do Escrow

Um job cron verifica a cada 5 minutos pagamentos em estado CAPTURED com approved_at + escrow_release_days <= NOW(). A liberação nunca é feita por ação direta do usuário — apenas pelo job do sistema ou pelo admin em casos excepcionais.

src/payment/jobs/escrow-release.job.ts
// Executa a cada 5 minutos via cron: */5 * * * *
export async function escrowReleaseJob() {
  const releaseDays = await PlatformSettings.get('escrow_release_days');

  // Busca pagamentos prontos para liberação (APPROVED + timer vencido)
  const candidates = await db.query(`
    SELECT p.id AS payment_id, o.id AS order_id
    FROM payments p
    JOIN orders o ON o.id = p.order_id
    WHERE p.status = 'CAPTURED'
      AND o.status = 'APPROVED'
      AND o.approved_at + ($1 || ' days')::INTERVAL <= NOW()
      AND NOT EXISTS (
        SELECT 1 FROM disputes d
        WHERE d.order_id = o.id AND d.status NOT IN ('CLOSED')
      )
    FOR UPDATE SKIP LOCKED   -- evita processamento duplo em múltiplas instâncias
    LIMIT 50                  -- processa em lotes para não sobrecarregar o gateway
  `, [releaseDays]);

  for (const row of candidates.rows) {
    try {
      await PaymentService.release(row.payment_id);
      logger.info({ paymentId: row.payment_id }, 'Escrow liberado');
    } catch (err) {
      // Falha isolada: não interrompe o lote. Será tentado novamente na próxima execução.
      logger.error({ paymentId: row.payment_id, err }, 'Falha ao liberar escrow');
      await alertOps(`Escrow stuck: ${row.payment_id}`);
    }
  }
}

static async refund({ paymentId, companyAmount, freelancerAmount, reason, adminId }) {
  const payment = await PaymentRepo.findById(paymentId);

  // Valida que os valores somam o gross (previne "criar dinheiro")
  const total = Decimal.add(companyAmount, freelancerAmount);
  if (Decimal.gt(total, payment.gross_amount))
    throw new BusinessError('REFUND_EXCEEDS_PAYMENT');

  const gateway = resolveGateway(payment.method);

  await db.transaction(async (trx) => {
    // Estorna para a empresa
    if (Decimal.gt(companyAmount, '0')) {
      await gateway.refund({ providerPaymentId: payment.provider_payment_id, amount: companyAmount });
      await TxLog.insert(trx, { paymentId, type: 'REFUND', status: 'SUCCESS', amount: companyAmount });
    }

    // Paga o freelancer (compensação parcial ou total)
    if (Decimal.gt(freelancerAmount, '0')) {
      const freelancer = await FreelancerRepo.findById(payment.freelancer_id);
      await gateway.transfer({
        amount:  freelancerAmount,
        pixKey:  freelancer.pix_key,
        orderId: payment.order_id,
      });
      await TxLog.insert(trx, { paymentId, type: 'RELEASE', status: 'SUCCESS', amount: freelancerAmount });
    }

    // Registra no ledger a perda de receita (se plataforma absorveu)
    const platformLoss = Decimal.sub(payment.gross_amount, total);
    if (Decimal.gt(platformLoss, '0')) {
      await trx.query(`
        INSERT INTO platform_ledger (payment_id, entry_type, amount, currency, balance_after, description)
        VALUES ($1,'REFUND_ISSUED',-$2,$3,
          (SELECT COALESCE(SUM(amount),0) FROM platform_ledger)-$2, $4)
      `, [paymentId, platformLoss, payment.currency, reason]);
    }

    await trx.query(`
      UPDATE payments SET status='REFUNDED', refunded_at=NOW() WHERE id=$1
    `, [paymentId]);
  });
}

9. Relação com Disputas

Quando uma disputa é aberta, o escrow é congelado — o job de liberação automática ignora pagamentos com disputas ativas. O admin tem controle total sobre a resolução, incluindo splits personalizados.

src/dispute/dispute.service.ts — resolve()
static async resolve({ disputeId, resolution, companyRefund, freelancerRelease, adminId }) {
  const dispute = await DisputeRepo.findById(disputeId);

  // Valida integridade: total deve ser <= gross_amount
  const payment    = await PaymentRepo.findByOrderId(dispute.order_id);
  const totalSplit  = Decimal.add(companyRefund, freelancerRelease);
  if (Decimal.gt(totalSplit, payment.gross_amount))
    throw new BusinessError('SPLIT_EXCEEDS_GROSS');

  // A taxa da plataforma é recalculada: absorve parte da perda em disputas
  const platformReceives = Decimal.sub(payment.gross_amount, totalSplit);

  switch (resolution) {
    case 'PAYMENT_RELEASED':    // Freelancer ganhou — libera tudo
      await PaymentService.release(payment.id);
      break;

    case 'REFUND_FULL':         // Empresa ganhou — estorna tudo
      await PaymentService.refund({
        paymentId:        payment.id,
        companyAmount:    payment.gross_amount,
        freelancerAmount: '0',
        reason:           'DISPUTE_RESOLVED_COMPANY',
        adminId,
      });
      break;

    case 'REFUND_PARTIAL':      // Split personalizado
    case 'SPLIT':
      await PaymentService.refund({
        paymentId:        payment.id,
        companyAmount:    companyRefund,
        freelancerAmount: freelancerRelease,
        reason:           `DISPUTE_RESOLVED_SPLIT:${adminId}`,
        adminId,
      });
      break;
  }

  await db.query(`
    UPDATE disputes SET status='CLOSED', resolved_by=$1, resolved_at=NOW(),
      resolution=$2, resolution_amount=$3, resolution_notes=$4
    WHERE id=$5
  `, [adminId, resolution, companyRefund, '...', disputeId]);
}

10. Reconciliação e Tratamento de Falhas

Um job diário compara o estado do sistema com o extrato do gateway. Detecta: pagamentos autorizados no gateway mas não no sistema, transferências que falharam silenciosamente, e discrepâncias de valor.

src/payment/jobs/reconciliation.job.ts
// Executa diariamente: 0 3 * * * (3h da manhã — baixo tráfego)
export async function reconciliationJob() {
  const yesterday = startOfDay(subDays(new Date(), 1));

  // 1. Busca todos os pagamentos do dia no sistema
  const systemPayments = await db.query(`
    SELECT provider_payment_id, status, gross_amount, provider
    FROM payments
    WHERE created_at >= $1 AND provider IS NOT NULL
  `, [yesterday]);

  // 2. Busca extrato do gateway no mesmo período
  const stripeCharges     = await stripe.charges.list({ created: { gte: unixTime(yesterday) }});
  const gerencianetPixs   = await efi.pixListCharges({ inicio: yesterday.toISOString() });

  // 3. Detecta discrepâncias
  const issues: string[] = [];

  for (const pay of systemPayments.rows) {
    const gatewayRecord = findInGateway(pay.provider_payment_id, stripeCharges);

    if (!gatewayRecord)
      issues.push(`MISSING_IN_GATEWAY: ${pay.provider_payment_id}`);
    else if (amountMismatch(pay.gross_amount, gatewayRecord.amount))
      issues.push(`AMOUNT_MISMATCH: ${pay.provider_payment_id} — system:${pay.gross_amount} gateway:${gatewayRecord.amount}`);
    else if (statusMismatch(pay.status, gatewayRecord.status))
      issues.push(`STATUS_MISMATCH: ${pay.provider_payment_id}`);
  }

  if (issues.length) {
    // Alerta o time financeiro e grava log para auditoria
    await alertFinancialTeam(issues);
    await saveReconciliationReport(yesterday, issues);
  }
}

11. Segurança Financeira

🔴 Nunca armazenar dados de cartão. A plataforma nunca vê o número do cartão — apenas o token gerado pelo Stripe.js no browser. Certificação PCI DSS é responsabilidade do Stripe. O sistema da plataforma é PCI SAQ A (mais leve).

  • Idempotency Keys obrigatórias em toda operação de pagamento. UUID gerado no cliente antes do request. Sem ela, a rota retorna 422. Persiste no banco com UNIQUE constraint.
  • HMAC timing-safe na validação de webhooks — usa crypto.timingSafeEqual para prevenir timing attacks que revelam a chave secreta comparando tempo de resposta.
  • Decimal arithmetic em todos os cálculos financeiros — nunca float. Use decimal.js ou Decimal nativo do PostgreSQL. 0.1 + 0.2 === 0.30000000000000004 em float causa centavos fantasmas.
  • Audit trail imutável em payment_transactions e platform_ledger. Nenhum UPDATE ou DELETE permitido para o role da aplicação. Todo estorno cria novo registro, não modifica o original.
  • Four-eyes para valores altos — pagamentos manuais acima de R$ 50.000 exigem aprovação de dois admins diferentes registrada no audit_log.
  • Rate limiting por empresa — máx 5 tentativas de pagamento por hora por empresa. Previne ataques de força bruta em cartões.
  • Webhook raw body — o Express deve ser configurado com express.raw() para a rota de webhook antes do express.json(). A assinatura HMAC é calculada sobre o body raw; parsear para JSON antes invalida a assinatura.
  • Credenciais via Secrets Manager — chaves de API do Stripe e Gerencianet nunca em variáveis de ambiente hardcoded. AWS Secrets Manager com rotação automática a cada 90 dias.
  • mTLS obrigatório para PIX — o BACEN exige que endpoints de webhook PIX usem mTLS com certificado emitido pela EFí. O Nginx termina o TLS e verifica o certificado do cliente (Gerencianet) antes de rotear para a aplicação.
Configuração Express para webhook (CRÍTICO)
// ERRADO — json() antes do raw() invalida a assinatura HMAC
app.use(express.json());
app.post('/webhooks/payment/stripe', stripeWebhookHandler);

// CORRETO — raw() apenas para as rotas de webhook
app.post('/webhooks/payment/stripe',
  express.raw({ type: 'application/json' }),  // raw antes do handler
  stripeWebhookHandler
);
app.use(express.json());  // json() para todas as outras rotas
