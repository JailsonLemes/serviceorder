Painel Administrativo

Back-office completo para gestão operacional da plataforma. RBAC com 6 papéis, auditoria imutável em toda ação, intervenções manuais com justificativa obrigatória e dashboards em tempo real.

1. Arquitetura do Painel

O painel admin é uma aplicação separada da plataforma principal — domínio diferente (admin.plataforma.com), autenticação própria com MFA obrigatório, e deploy independente. Nenhuma rota de admin fica exposta na API pública.

2. Papéis e Permissões (RBAC)

Seis papéis com permissões granulares por recurso e ação. Nenhuma rota de admin aceita token de usuário comum — o JWT de admin usa uma chave de assinatura separada e inclui o claim role.

export const adminGuard = (permission: string) =>
  async (req: Request, res: Response, next: NextFunction) => {
    // JWT assinado com chave exclusiva do admin (RS256_ADMIN_KEY)
    const token  = extractBearer(req);
    const claims = verifyAdminJwt(token);  // lança se inválido

    // Verifica se o role possui a permissão
    if (!hasPermission(claims.role, permission))
      return res.status(403).json({ error: 'INSUFFICIENT_PERMISSIONS', required: permission });

    // Verifica que o admin não está suspenso (verificação no Redis, TTL 5min)
    const adminStatus = await getCachedAdminStatus(claims.sub);
    if (adminStatus !== 'ACTIVE')
      return res.status(401).json({ error: 'ADMIN_ACCOUNT_INACTIVE' });

    req.admin = { id: claims.sub, role: claims.role, email: claims.email };
    next();
  };

// Tabela de permissões por role
const ROLE_PERMISSIONS: Record<string, string[]> = {
  SUPER_ADMIN: ['*'],  // wildcard: acesso total
  FINANCIAL_ADMIN: [
    'users:read', 'orders:read',
    'payments:read', 'payments:manual_release', 'payments:refund', 'payments:reconcile',
    'audit:read',
  ],
  SUPPORT_ADMIN: [
    'users:read', 'users:suspend', 'users:verify_identity',
    'orders:read', 'orders:cancel', 'orders:extend_deadline',
    'disputes:assign', 'disputes:resolve',
    'ratings:remove',
  ],
  CONTENT_ADMIN: ['categories:read', 'categories:write'],
  ANALYST:       ['users:read', 'orders:read', 'payments:read', 'audit:read'],
  COMPLIANCE:    ['users:read', 'users:suspend', 'orders:read', 'payments:read', 'audit:read'],
};

function hasPermission(role: string, permission: string): boolean {
  const perms = ROLE_PERMISSIONS[role] ?? [];
  return perms.includes('*') || perms.includes(permission);
}

3. Módulo de Usuários

Toda ação mutante exige justificativa obrigatória que vai para o audit_log. Impersonation gera sessão auditada separada — o admin vê a plataforma como o usuário sem saber a senha dele.

src/admin/modules/admin-user.service.ts
export class AdminUserService {

  // ── Busca com filtros avançados ────────────────────────────────────────
  static async search(filters: {
    query?:       string;    // busca em nome, email, CPF, CNPJ
    role?:        'EMPRESA' | 'FREELANCER';
    status?:      'ACTIVE' | 'SUSPENDED' | 'PENDING_VERIFICATION';
    hasDisputes?: boolean;
    createdFrom?: string;
    createdTo?:   string;
    cursor?:      string;    // paginação por cursor
    limit?:       number;
  }) {
    return db.query(`
      SELECT u.*, f.reputation_score, f.completion_rate,
        (SELECT COUNT(*) FROM disputes d JOIN orders o ON o.id=d.order_id
         WHERE o.selected_freelancer_id=f.id OR o.company_id=c.id) AS dispute_count
      FROM users u
      LEFT JOIN freelancers f ON f.user_id=u.id
      LEFT JOIN companies  c ON c.user_id=u.id
      WHERE ($1 IS NULL OR u.search_vector @@ plainto_tsquery('portuguese',$1))
        AND ($2 IS NULL OR u.role=$2)
        AND ($3 IS NULL OR u.status=$3)
        AND ($4 IS NULL OR u.created_at >= $4)
        AND ($5 IS NULL OR u.created_at <= $5)
        AND ($6 IS NULL OR u.id > $6)
      ORDER BY u.id LIMIT $7
    `, [filters.query, filters.role, filters.status,
       filters.createdFrom, filters.createdTo,
       filters.cursor, filters.limit ?? 50]);
  }

  // ── Suspender usuário ─────────────────────────────────────────────────
  static async suspend(targetUserId: string, reason: string, admin: AdminContext) {
    const before = await UserRepo.findById(targetUserId);

    await db.transaction(async (trx) => {
      await trx.query(`
        UPDATE users SET status='SUSPENDED', suspended_at=NOW(), suspension_reason=$1
        WHERE id=$2
      `, [reason, targetUserId]);

      // Invalida todos os tokens de sessão do usuário imediatamente
      await trx.query(`UPDATE refresh_tokens SET revoked_at=NOW() WHERE user_id=$1`, [targetUserId]);
      await redis.del(`session:${targetUserId}:*`);  // Redis SCAN + DEL por padrão

      await AuditLog.write(trx, {
        adminId:    admin.id,
        action:     'user.suspend',
        entityType: 'user',  entityId: targetUserId,
        reason,
        beforeData: { status: before.status },
        afterData:  { status: 'SUSPENDED' },
      });
    });
  }

  // ── Impersonação (session auditada) ───────────────────────────────────
  static async impersonate(targetUserId: string, admin: AdminContext) {
    // Gera token de curta duração (15min, não renovável) com claim de impersonação
    const impersonateToken = signJwt({
      sub:          targetUserId,
      impersonatedBy: admin.id,
      exp:          Math.floor(Date.now() / 1000) + 900,
    });
    await AuditLog.write(null, {
      adminId: admin.id, action: 'user.impersonate',
      entityType: 'user', entityId: targetUserId,
    });
    return { token: impersonateToken, redirectTo: process.env.APP_URL };
  }
}

4. Módulo de Ordens — Intervenção Manual

src/admin/modules/admin-order.service.ts
export class AdminOrderService {

  // ── Força cancelamento com motivo admin ────────────────────────────────
  static async forceCancel(orderId: string, reason: string, admin: AdminContext) {
    const order   = await OrderRepo.findById(orderId);
    const TERMINAL = ['COMPLETED', 'CANCELLED'];
    if (TERMINAL.includes(order.status))
      throw new BusinessError('ORDER_ALREADY_TERMINAL');

    await db.transaction(async (trx) => {
      await trx.query(`
        UPDATE orders SET status='CANCELLED', cancelled_at=NOW(),
          cancel_reason=$1, cancelled_by_admin=$2
        WHERE id=$3
      `, [reason, admin.id, orderId]);

      // Se havia pagamento capturado, devolve à empresa automaticamente
      const payment = await PaymentRepo.findByOrder(orderId);
      if (payment && ['AUTHORIZED','CAPTURED','DISPUTED'].includes(payment.status))
        await queue.publish('payment.admin_refund', {
          paymentId: payment.id, adminId: admin.id, reason,
        });

      await AuditLog.write(trx, {
        adminId: admin.id, action: 'order.force_cancel',
        entityType: 'order', entityId: orderId, reason,
        beforeData: { status: order.status },
        afterData:  { status: 'CANCELLED' },
      });
    });
  }

  // ── Estende prazo com notificação às partes ────────────────────────────
  static async extendDeadline(orderId: string, newDeadline: Date, reason: string, admin: AdminContext) {
    const before = await OrderRepo.findById(orderId);
    await db.query(`UPDATE orders SET deadline_at=$1 WHERE id=$2`, [newDeadline, orderId]);
    await queue.publish('order.deadline_extended', { orderId, newDeadline, reason });
    await AuditLog.write(null, {
      adminId: admin.id, action: 'order.extend_deadline',
      entityType: 'order', entityId: orderId, reason,
      beforeData: { deadline_at: before.deadline_at },
      afterData:  { deadline_at: newDeadline },
    });
  }
}

5. Módulo de Pagamentos

src/admin/modules/admin-payment.service.ts
export class AdminPaymentService {

  // ── Liberar escrow manualmente (ex: bug no job automático) ─────────────
  static async manualRelease(paymentId: string, reason: string, admin: AdminContext) {
    const payment = await PaymentRepo.findById(paymentId);

    if (payment.status !== 'CAPTURED')
      throw new BusinessError('PAYMENT_NOT_CAPTURED');

    // Valores acima de R$50k exigem segundo admin (four-eyes)
    if (Number(payment.gross_amount) > 50000 && !admin.secondApprovalId)
      throw new BusinessError('SECOND_APPROVAL_REQUIRED');

    await PaymentService.release(paymentId);
    await AuditLog.write(null, {
      adminId: admin.id, action: 'payment.manual_release',
      entityType: 'payment', entityId: paymentId, reason,
      afterData: { amount: payment.freelancer_amount, secondApproval: admin.secondApprovalId },
    });
  }

  // ── Estorno administrativo (fora do fluxo de disputa) ─────────────────
  static async adminRefund(input: { paymentId: string; amount?: string; reason: string }, admin: AdminContext) {
    await PaymentService.refund({
      paymentId:        input.paymentId,
      companyAmount:    input.amount ?? 'FULL',
      freelancerAmount: '0',
      reason:           `ADMIN_REFUND:${admin.id}:${input.reason}`,
      adminId:          admin.id,
    });
    await AuditLog.write(null, {
      adminId: admin.id, action: 'payment.admin_refund',
      entityType: 'payment', entityId: input.paymentId,
      reason: input.reason,
    });
  }

  // ── Visão do ledger (receita contábil da plataforma) ───────────────────
  static async getLedgerSummary(from: Date, to: Date) {
    return db.query(`
      SELECT
        entry_type,
        SUM(amount)   AS total,
        COUNT(*)      AS count,
        currency
      FROM platform_ledger
      WHERE created_at BETWEEN $1 AND $2
      GROUP BY entry_type, currency
      ORDER BY entry_type
    `, [from, to]);
  }
}

6. Auditoria — Log Imutável

Toda ação administrativa grava no audit_logs. A tabela tem REVOKE UPDATE, DELETE ON audit_logs FROM app_role no DDL — nem a própria aplicação pode apagar registros. Apenas o role audit_reader (readonly, sem acesso à aplicação) pode ler.

src/admin/audit/audit-log.service.ts
export class AuditLog {
  static async write(
    trx: Transaction | null,
    entry: {
      adminId:    string;
      action:     string;       // ex: 'user.suspend', 'payment.manual_release'
      entityType: string;
      entityId:   string;
      reason?:    string;
      beforeData?: unknown;
      afterData?:  unknown;
    }
  ) {
    const conn = trx ?? db;
    await conn.query(`
      INSERT INTO audit_logs
        (admin_id, action, entity_type, entity_id, reason,
         before_data, after_data, ip_address, user_agent, created_at)
      VALUES ($1,$2,$3,$4,$5,$6,$7,$8,$9,NOW())
    `, [
      entry.adminId,  entry.action,
      entry.entityType, entry.entityId,
      entry.reason ?? null,
      entry.beforeData ? JSON.stringify(entry.beforeData) : null,
      entry.afterData  ? JSON.stringify(entry.afterData) : null,
      getRequestIP(),  getRequestUA(),
    ]);
  }

  // Busca com filtros — usada pelo módulo Audit Logs do painel
  static async query(filters: {
    adminId?:    string;
    action?:     string;
    entityType?: string;
    entityId?:   string;
    from?:       string;
    to?:         string;
    cursor?:     string;
  }) {
    return db.query(`
      SELECT a.*, adm.email AS admin_email, adm.role AS admin_role
      FROM audit_logs a
      JOIN admins adm ON adm.id = a.admin_id
      WHERE ($1 IS NULL OR a.admin_id = $1)
        AND ($2 IS NULL OR a.action LIKE $2 || '%')
        AND ($3 IS NULL OR a.entity_type = $3)
        AND ($4 IS NULL OR a.entity_id = $4)
        AND ($5 IS NULL OR a.created_at >= $5)
        AND ($6 IS NULL OR a.created_at <= $6)
        AND ($7 IS NULL OR a.id > $7)
      ORDER BY a.created_at DESC LIMIT 100
    `, [filters.adminId, filters.action, filters.entityType,
       filters.entityId, filters.from, filters.to, filters.cursor]);
  }
}

7. Gestão de Categorias, Skills e Taxas

src/admin/modules/admin-settings.service.ts
export class AdminSettingsService {

  // ── Cria/edita categoria (hierarquia em árvore) ────────────────────────
  static async upsertCategory(input: {
    id?:         string;
    name:        string;
    parentId?:   string;   // null = categoria raiz
    description?: string;
    iconUrl?:    string;
    isActive:    boolean;
  }, admin: AdminContext) {
    // Valida que parentId não cria ciclo na árvore
    if (input.parentId && input.id)
      await assertNoCycle(input.id, input.parentId);

    const result = await db.query(`
      INSERT INTO categories (id, name, parent_id, description, icon_url, is_active, slug)
      VALUES (COALESCE($1, gen_random_uuid()), $2, $3, $4, $5, $6,
              slugify($2))
      ON CONFLICT (id) DO UPDATE
        SET name=$2, parent_id=$3, description=$4, icon_url=$5,
            is_active=$6, updated_at=NOW()
      RETURNING *
    `, [input.id, input.name, input.parentId,
       input.description, input.iconUrl, input.isActive]);

    await AuditLog.write(null, {
      adminId: admin.id, action: 'category.upsert',
      entityType: 'category', entityId: result.rows[0].id,
      afterData: input,
    });
    await redis.del('categories:tree');   // invalida cache da árvore de categorias
    return result.rows[0];
  }

  // ── Atualiza configuração de plataforma ────────────────────────────────
  static async updateSetting(key: string, value: string, admin: AdminContext) {
    const ALLOWED_KEYS = [
      'platform_fee_rate',      // taxa padrão (ex: '0.10')
      'escrow_release_days',     // dias para liberar após aprovação
      'rating_window_hours',     // janela de avaliação (ex: '72')
      'dispute_sla_hours',       // SLA de resolução de disputas
      'max_adjustment_cycles',   // máx ciclos de revisão
      'pix_expiration_seconds',  // expiração do QR code PIX
    ];
    if (!ALLOWED_KEYS.includes(key))
      throw new ValidationError('SETTING_KEY_NOT_ALLOWED');

    const before = await PlatformSettings.get(key);
    await db.query(`
      INSERT INTO platform_settings (key, value, updated_by)
      VALUES ($1, $2, $3)
      ON CONFLICT (key) DO UPDATE SET value=$2, updated_by=$3, updated_at=NOW()
    `, [key, value, admin.id]);
    await redis.del(`setting:${key}`);   // invalida cache imediatamente
    await AuditLog.write(null, {
      adminId: admin.id, action: `setting.update`,
      entityType: 'setting', entityId: key,
      beforeData: { value: before },  afterData: { value },
    });
  }

  // ── Taxa customizada por empresa (enterprise) ──────────────────────────
  static async setCompanyFeeRate(companyId: string, rate: string, admin: AdminContext) {
    const rateNum = Number(rate);
    if (rateNum < 0 || rateNum > 1)  throw new ValidationError('RATE_OUT_OF_RANGE');
    await db.query(`UPDATE companies SET custom_fee_rate=$1 WHERE id=$2`, [rate, companyId]);
    await AuditLog.write(null, {
      adminId: admin.id, action: 'company.set_fee_rate',
      entityType: 'company', entityId: companyId,
      afterData: { custom_fee_rate: rate },
    });
  }
}

8. Dashboard — Métricas e KPIs

src/admin/modules/admin-dashboard.service.ts
export class AdminDashboardService {
  static async getOverview(period: '7d' | '30d' | '90d') {
    const cacheKey = `admin:dashboard:${period}`;
    const cached   = await redis.get(cacheKey);
    if (cached) return JSON.parse(cached);

    const days = { '7d':7, '30d':30, '90d':90 }[period];

    const [financial, orders, users, disputes, sla] = await Promise.all([

      // GMV + Receita
      db.query(`
        SELECT
          SUM(gross_amount)         AS gmv,
          SUM(platform_fee_amount)  AS revenue,
          AVG(gross_amount)         AS avg_order_value,
          SUM(CASE WHEN status='REFUNDED' THEN gross_amount ELSE 0 END) AS refunded
        FROM payments
        WHERE created_at >= NOW() - ($1 || ' days')::INTERVAL
      `, [days]),

      // Ordens por status + taxa de conclusão
      db.query(`
        SELECT status, COUNT(*) AS n FROM orders
        WHERE created_at >= NOW() - ($1 || ' days')::INTERVAL
        GROUP BY status
      `, [days]),

      // Novos usuários
      db.query(`
        SELECT role, COUNT(*) AS n FROM users
        WHERE created_at >= NOW() - ($1 || ' days')::INTERVAL
        GROUP BY role
      `, [days]),

      // Taxa e tempos de disputa
      db.query(`
        SELECT
          COUNT(*)                                          AS total,
          COUNT(*) FILTER (WHERE status='CLOSED')          AS resolved,
          AVG(EXTRACT(EPOCH FROM (resolved_at-created_at))/3600)
            FILTER (WHERE resolved_at IS NOT NULL)         AS avg_resolution_hours,
          COUNT(*) FILTER (WHERE status='ESCALATED')       AS escalated
        FROM disputes
        WHERE created_at >= NOW() - ($1 || ' days')::INTERVAL
      `, [days]),

      // Disputas em SLA breach agora
      db.query(`
        SELECT COUNT(*) AS breach_count FROM disputes
        WHERE status IN ('OPEN','UNDER_REVIEW')
          AND expected_resolution_at < NOW()
      `),
    ]);

    const result = {
      financial: financial.rows[0],
      orders:    orders.rows,
      users:     users.rows,
      disputes:  disputes.rows[0],
      slaBreaches: Number(sla.rows[0].breach_count),
      generatedAt: new Date().toISOString(),
    };

    // Cache por 2 minutos — dados quase em tempo real
    await redis.setEx(cacheKey, 120, JSON.stringify(result));
    return result;
  }
}

10. Segurança do Painel

  • MFA obrigatório: TOTP (Google Authenticator / Authy) exigido em todos os logins de admin. Sem MFA, conta não é ativada.
  • IP allowlist: logins de admin só aceitos de IPs cadastrados. Novo IP exige aprovação por email.
  • JWT de admin separado: assinado com RS256_ADMIN_PRIVATE_KEY diferente da chave pública. Token de usuário não funciona em rotas /admin/*.
  • Session timeout: sessão de admin expira em 4 horas. Não é renovável — requer novo login com MFA.
  • Four-eyes para operações críticas: liberações manuais acima de R$ 50k e mudanças em platform_fee_rate exigem aprovação de segundo admin com log separado.
  • Rate limiting agressivo: máximo 100 requests/min por admin. Login: máximo 5 tentativas, depois bloqueio de 30min + alerta.
  • Audit de tudo: leitura de dados sensíveis (impersonation, visualizar CPF) também é logada, não apenas mutações.
  • Separação de deploy: painel admin em subdomínio separado com CSP restritiva, sem assets compartilhados com a plataforma pública.

⚠️ Impersonation é a ação de maior risco. Gera token de 15 minutos não renovável, exige justificativa obrigatória, e toda ação feita durante a sessão é registrada com flag impersonated_by: admin_id. O usuário impersonado recebe notificação por email após o término da sessão.
