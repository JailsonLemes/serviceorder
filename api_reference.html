API Reference

Documentação completa da API REST do marketplace de ordens de serviço. Todas as respostas são JSON. Datas em ISO 8601 UTC. Valores monetários como string decimal com 2 casas.

Base URL https://api.seudominio.com/v1
Convenções e Padrões

Envelope de resposta

// Sucesso
{
  "data": { /* payload */ },
  "meta": { /* paginação */ }
}

// Erro
{
  "error": {
    "code": "INVALID_CREDENTIALS",
    "message": "E-mail ou senha inválidos",
    "details": []
  }
}

Paginação cursor

// Query params
?page=1&per_page=20
&sort_by=created_at&sort_dir=desc

// Meta na resposta
"meta": {
  "page": 1,
  "per_page": 20,
  "total": 156,
  "total_pages": 8
}

Autenticação JWT

// Header obrigatório em rotas protegidas
Authorization: Bearer {access_token}

// Tokens
access_token  → 15 min (RS256)
refresh_token → 30 dias (HttpOnly cookie)

Idempotência (pagamentos)

// Header obrigatório em operações financeiras
X-Idempotency-Key: 7f3d4a28-1c9e-4b22-a8f6

// UUID gerado pelo CLIENTE antes do request.
// Mesmo key = mesma resposta, sem dupla cobrança.

HTTPCódigoQuando ocorre

POST /auth/refresh
Público
Renova o access token. Aceita o refresh_token via body ou via cookie HttpOnly. O token antigo é invalidado (token rotation).
Request Body
{
  "refresh_token": "dGhpcyBpcyBhIHJlZnJlc2ggdG9rZW4..."   // ou via cookie — um dos dois
}
Response 200
{
  "data": {
    "access_token":  "eyJhbGciOiJSUzI1NiJ9.novo_token...",
    "refresh_token": "novo_refresh_token...",
    "expires_in":    900
  }
}
Erros específicos
HTTPCódigoCausa
401REFRESH_TOKEN_INVALIDToken expirado, revogado, reutilizado (token rotation) ou inexistente
POST /auth/logout
🔒 Autenticado
Revoga o refresh token informado. Invalida a sessão imediatamente — o access token expira naturalmente em até 15 min.
Request Body

Response 204
// Sem body

POST /orders/:order_id/dispute/messages
🔒 Participante ou Admin
Adiciona mensagem à disputa. Admins podem enviar mensagens internas (is_internal: true) visíveis somente entre admins.
Request Body
{
  "content":     "Estou enviando o log de erro completo do servidor de produção para comprovar o problema.",
  "is_internal": false   // true apenas para ADMIN — notas internas do time de suporte
}
Response 201
{
  "data": {
    "id":          "dmsg-uuid-novo",
    "content":    "Estou enviando o log de erro...",
    "is_internal": false,
    "created_at": "2025-05-20T19:00:00Z"
  }
}

POST /disputes/:dispute_id/assign
Admin
Admin se auto-atribui ou atribui a outro admin. Muda status para UNDER_REVIEW.
Request Body
{ "admin_id": "admin-uuid" }
Response 200
{ "data": { "dispute_id": "disp-uuid", "status": "UNDER_REVIEW", "assigned_to": "admin-uuid" } }
POST /disputes/:dispute_id/resolve
Admin
Admin resolve a disputa. A decisão determina o destino do escrow. Todas as transações financeiras são processadas atomicamente dentro de uma única transação de banco.
Request Body
{
  "resolution":         "REFUND_PARTIAL",   // REFUND_FULL | REFUND_PARTIAL | PAYMENT_RELEASED | SPLIT | NO_ACTION
  "resolution_amount":  "1750.00",          // obrigatório para REFUND_PARTIAL e SPLIT
  "resolution_notes":   "Após análise das evidências de ambas as partes, fica claro que o freelancer entregou aproximadamente 50% do escopo acordado com qualidade adequada. Decisão: reembolso de 50% para a empresa, liberação de 50% para o freelancer descontada a taxa da plataforma."
}
Response 200
{
  "data": {
    "dispute_id":          "disp-uuid-disputa",
    "status":              "CLOSED",
    "resolution":          "REFUND_PARTIAL",
    "company_refund":       "1750.00",
    "freelancer_payment":  "1400.00",           // 1750 - 20% plataforma (após disputa)
    "platform_fee":        "350.00",
    "order_final_status":  "CANCELLED",         // CANCELLED | COMPLETED | IN_PROGRESS
    "resolved_at":         "2025-05-22T10:00:00Z"
  }
}

GET /search/services
Freelancer
Busca full-text em ordens PUBLISHED e IN_APPLICATION via Elasticsearch. Suporta filtros avançados, geolocalização e ordenação por relevância.
Query Parameters
ParâmetroTipoDescrição
qopcionalstringTexto de busca (min 2 chars)
category_idsopcionalstringUUIDs separados por vírgula
skill_idsopcionalstringSkills requeridas (match ANY)
budget_min / budget_maxopcionaldecimalFaixa de orçamento
location_typeopcionalstringREMOTE | ON_SITE | HYBRID
lat / lng / radius_kmopcionaldecimal / integerBusca geoespacial para ON_SITE (padrão: 50km)
sort_byopcionalstringrelevance (padrão) | created_at | budget_max | deadline_at
page / per_pageopcionalintegerPaginação — padrão 1 / 20
Response 200
{
  "data": [ /* array de ordens no mesmo formato de GET /orders */ ],
  "meta": {
    "page": 1, "per_page": 20, "total": 203,
    "query_time_ms": 18
  }
}
